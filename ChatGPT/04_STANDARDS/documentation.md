# Documentation Standards

## Hierarchy

1.  **README**: The entry point for every folder. Must explain _Purpose_ and _Contents_.
2.  **Context**: `context.md` for projects. Explains the _Why_.
3.  **Code Comments**: For complex logic only.

## Format

- **Markdown**: The universal standard.
- **Links**: Use relative paths for portability.
- **Diagrams**: Use Mermaid for flows and structures.

## Requirements

- **Update on Change**: Code and Docs must stay in sync.
- **Explicit Ownership**: Every doc must have a clear owner (Role).
- **Date Stamped**: Major updates should be logged.
